## Описання проекту

**Pyrih's Garage** — це повнофункціональна веб-система управління автосервісом (станцією технічного обслуговування), що дозволяє:

### Функціональність для клієнтів:
-  **Онлайн-бронювання послуг** з вибором майстра, дати та часу
-  **Профіль користувача** з реєстрацією та авторизацією
-  **Керування гаражем** — додавання та видалення автомобілів
-  **Історія замовлень** з кодами підтвердження
-  **Сповіщення** про статус замовлень

### Функціональність для адміністратора:
-  **Адмін-панель** з аналітикою та статистикою
-  **Управління майстрами** — додавання, редагування, видалення
-  **Управління послугами** — налаштування прайс-листу
-  **Управління замовленнями** — зміна статусу, видалення
-  **Графіки та звіти** дохідності і завантаженості

### Технічні можливості:
-  **Темна тема** — автоматична зміна за часом суток
-  **Адаптивний дизайн** — повна сумісність з мобільними пристроями
-  **JWT-автентифікація** з хешуванням паролів (bcrypt)
-  **SQLite база даних** з підтримкою основних операцій CRUD
-  **Express.js сервер** з CORS та REST API

---

### Стек технологій

**Backend:**
- Node.js + Express.js
- SQLite (better-sqlite3)
- JWT (jsonwebtoken)
- Bcrypt для хешування паролів
- CORS для кросс-доменних запитів

**Frontend:**
- HTML5 + CSS3 + Vanilla JavaScript (ES6+)
- Chart.js для графіків
- Flatpickr для вибору дат
- SPA архітектура (Single Page Application)

**Інші:**
- Git для контролю версій
- ER-діаграма БД (ER_Diagram.html)

## Основні компоненти

### Backend API

**Базова URL:** `http://localhost:5500/api`

#### Авторизація
- `POST /auth/register` — реєстрація нового користувача
- `POST /auth/login` — вхід у систему
- `GET /auth/me` — отримання поточного користувача

#### Користувачі
- `GET /users` — список всіх користувачів (адмін)
- `PUT /users/:id` — редагування користувача

#### Автомобілі
- `GET /cars` — список автомобілів поточного користувача
- `POST /cars` — додавання автомобіля
- `PUT /cars/:id` — редагування автомобіля
- `DELETE /cars/:id` — видалення автомобіля

#### Замовлення
- `GET /bookings` — список замовлень
- `POST /bookings` — створення замовлення
- `PUT /bookings/:id` — редагування статусу замовлення
- `DELETE /bookings/:id` — видалення замовлення

#### Майстри
- `GET /masters` — список майстрів
- `POST /masters` — додавання майстра (адмін)
- `PUT /masters/:id` — редагування майстра (адмін)
- `DELETE /masters/:id` — видалення майстра (адмін)

#### Послуги
- `GET /services` — список послуг
- `POST /services` — додавання послуги (адмін)
- `PUT /services/:id` — редагування послуги (адмін)
- `DELETE /services/:id` — видалення послуги (адмін)

#### Аналітика
- `GET /analytics/stats` — загальна статистика (адмін)

### Frontend модулі

**auth.js** — Управління автентифікацією:
- Реєстрація та вхід користувача
- Зберігання JWT токену
- Перевірка статусу входу

**api.js** — HTTP клієнт:
- Методи для REST запитів
- Обробка помилок
- Управління токенами

**booking.js** — Система бронювання:
- Вибір послуги і майстра
- Вибір дати та часу
- Введення деталей автомобіля
- Підтвердження замовлення

**garage.js** — Управління автомобілями:
- Додавання нових автомобілів
- Редагування даних автомобіля
- Видалення з гаража

**admin.js** — Адмін-функціональність:
- Управління майстрами
- Управління послугами
- Управління замовленнями
- Просмотр статистики

## База даних

### Схема БД

**Таблиця `users`**
- `id` — первинний ключ
- `name`, `sname` — ім'я та прізвище
- `email` — унікальна адреса (логін)
- `phone` — номер телефону
- `pass_hash` — хешований пароль (bcrypt)
- `role` — роль ('client' або 'admin')
- `created_at` — дата реєстрації

**Таблиця `cars`**
- `id` — первинний ключ
- `user_id` — посилання на користувача
- `make`, `model` — марка і модель
- `plate` — номерний знак
- `year`, `vin`, `mileage`, `color` — технічні характеристики

**Таблиця `bookings`**
- `id` — первинний ключ
- `code` — унікальний код замовлення
- `svc_id`, `svc_name`, `price` — інформація про послугу
- `master` — ім'я майстра
- `date`, `time` — дата і час
- `car`, `plate` — інформація про автомобіль
- `comment` — коментар клієнта
- `client_id`, `client_email` — контактні дані
- `status` — статус замовлення

**Таблиця `masters`**
- `id` — первинний ключ
- `name` — ім'я майстра
- `spec` — спеціалізація
- `rating` — рейтинг
- `phone` — контакт

**Таблиця `services`**
- `id` — первинний ключ
- `name` — назва послуги
- `price` — вартість
- `dur` — приблизна тривалість
- `svg` — SVG іконка

## Користування системою

### Для звичайного клієнта

1. **Реєстрація:**
   - Клацніть "Увійти"
   - Виберіть "Реєстрація"
   - Заповніть форму (ім'я, прізвище, email, пароль, телефон)
   - Підтвердіть дані

2. **Бронювання послуги:**
   - Перейдіть на вкладку "Запис"
   - Виберіть послугу з доступних варіантів
   - Виберіть майстра та його вільний слот
   - Введіть дату, час та дані автомобіля
   - Додайте коментар за необхідності
   - Підтвердіть замовлення
   - Отримайте код підтвердження

3. **Керування гаражем:**
   - Перейдіть на вкладку "Мій гараж"
   - Додайте нові автомобілі
   - Редагуйте дані своїх машин
   - Видаліть машини за потребою

### Для адміністратора

1. **Доступ до адмін-панелі:**
   - Увійдіть як адміністратор
   - На навбар з'явиться вкладка "Адмінка"

2. **Управління послугами:**
   - Переглядайте список послуг
   - Додавайте нові послуги з ціною та тривалістю
   - Редагуйте існуючі послуги
   - Видаляйте непотрібні послуги

3. **Управління майстрами:**
   - Керуйте списком майстрів
   - Додавайте нових фахівців
   - Встановлюйте рейтинги і спеціалізацію
   - Видаляйте майстрів при необхідності

4. **Управління замовленнями:**
   - Переглядайте всі замовлення клієнтів
   - Змінюйте статус замовлень
   - Видаляйте скасовані замовлення

5. **Аналітика:**
   - Переглядайте графіки дохідності
   - Аналізуйте завантаженість майстрів
   - Вивчайте статистику за період

## Безпека

### Реалізовані заходи

- ✅ **JWT токени** для безпечної автентифікації
- ✅ **Bcrypt хешування** для паролів (10 раундів солі)
- ✅ **CORS** для контролю доступу до API
- ✅ **Валідація вхідних даних** на серверу
- ✅ **Уникнення SQL-injection** через параметризовані запити
- ✅ **Розділення ролей** (client/admin) з перевіркою на серверу
- ✅ **HTTPS готовність** (легко налаштовується на продакшн-серверу)

### Рекомендації для продакшну

1. Змініть `SECRET` у файлі `backend/server.js` на надійне значення
2. Налаштуйте HTTPS сертифікати
3. Встановіть змінні окруження для конфіденційних даних
4. Використовуйте обернене проксі (nginx/Apache)
5. Налаштуйте логування і моніторинг
6. Регулярно оновлюйте залежності npm

## Розпізнавання проблем та їх вирішення

### Проблема: "Cannot find module 'better-sqlite3'"
**Рішення:** Переустановіть залежності
```bash
npm install
```

### Проблема: Порт 5500 вже зайнятий
**Рішення:** Змініть PORT у файлі `backend/server.js`

### Проблема: Стиль не застосовується
**Рішення:** Очистіть кеш браузера (Ctrl+Shift+Delete)

### Проблема: Авторизація не працює
**Рішення:** Перевірте, чи браузер підтримує cookies, і чи не блокується JavaScript
